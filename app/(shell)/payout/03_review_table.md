# Review Step — Layout

The Review screen contains three primary sections:

1. Workflow Stepper
2. Run Summary Card
3. Review Table

---

## Workflow Stepper

The payout workflow uses a three-step progress indicator.

Steps:

1. Review
2. Exceptions
3. Approve

Example:

● Review      ○ Exceptions      ○ Approve

Step states:

Completed
✔ icon

Active
● icon

Upcoming
○ icon

Examples:

Review step:

● Review      ○ Exceptions      ○ Approve

Exceptions step:

✔ Review      ● Exceptions      ○ Approve

Approve step:

✔ Review      ✔ Exceptions      ● Approve

---

## Step Action Button

The primary step action appears aligned to the right of the stepper.

Examples:

Review step
Continue to Exceptions →

Exceptions step
Continue to Approval →

Approve step
Approve payout →

The action button advances the payout workflow.

---

## View Summary Card

The View Summary Card provides a quick snapshot of the payout run.

It appears below the stepper and above the review table.

Metrics include:

Professionals
Activity records
Total payout
Issues detected
Confidence

Example:

12 Professionals | 2,140 Activity records | $8,225 Total payout | 0 Issues | High Confidence

Purpose:

The view summary allows Clinical Ops to confirm the payout run appears correct before reviewing individual payouts.
