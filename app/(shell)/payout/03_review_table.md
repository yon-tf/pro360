# Review Steps 1 and 2 — TFP and Hotline Review Layout

This doc covers the first two working steps of the payout review experience:

1. Review TFP
2. Review Hotline

The full workflow still has four primary steps:

1. Review TFP
2. Review Hotline
3. Exceptions
4. Approve

---

## Workflow Stepper

The payout workflow uses a four-step progress indicator.

Steps:

1. Review TFP
2. Review Hotline
3. Exceptions
4. Approve

Example:

● Review TFP      ○ Review Hotline      ○ Exceptions      ○ Approve

Step states:

Completed
✔ icon

Active
● icon

Upcoming
○ icon

Examples:

Review TFP step:

● Review TFP      ○ Review Hotline      ○ Exceptions      ○ Approve

Review Hotline step:

✔ Review TFP      ● Review Hotline      ○ Exceptions      ○ Approve

Exceptions step:

✔ Review TFP      ✔ Review Hotline      ● Exceptions      ○ Approve

Approve step:

✔ Review TFP      ✔ Review Hotline      ✔ Exceptions      ● Approve

---

## Step Action Button

The primary step action appears aligned to the right of the stepper.

Examples:

Review TFP step
Continue to Hotline →

Review Hotline step
Continue to Exceptions →

Exceptions step
Continue to Approve →

Approve step
Approve payout →

The action button advances the payout workflow.

---

## View Summary Card

The View Summary Card provides a quick snapshot of the payout run.

It appears below the stepper and above the review table.

Metrics include:

Professionals
Activity records
Total payout
Issues detected
Confidence

Example:

12 Professionals | 2,140 Activity records | $8,225 Total payout | 0 Issues | High Confidence

Purpose:

The view summary allows Clinical Ops to confirm the payout run appears correct before reviewing individual payouts.
